Estructura básica HTML5

HTML, significa Lenguaje de Marcado para Hipertextos (HyperText Markup Language), es el lenguaje utilizado para describir y definir el contenido de una Página Web en un formato bien estructurado. Es el elemento de construcción más básico de una página web y se usa para crear y representar visualmente una página web. Determina el contenido de la página web, pero no su funcionalidad. Otras tecnologías distintas de HTML son usadas generalmente para describir la apariencia/presentación de una página web (CSS) o su funcionalidad (JavaScript).

HTML le da "valor añadido" a un texto estándar en español. Hiper Texto se refiere a enlaces que conectan una página Web con otra, ya sea dentro de una página web o entre diferentes sitios web. Los vínculos son un aspecto fundamental de la Web. Al subir contenido a Internet y vincularlo a páginas de otras personas, te haces participante activo de esta Red Mundial.

HTML usa "markup" o marcado para anotar textos, imágenes, y otros contenidos que se muestran en el Navegador Web. El lenguaje de marcado HTML incluye "elementos" especiales tales como <head>, <title>, <body>, <header>, <article>, <section>, <p>, <div>, <span>, <img>, y muchos otros más.

Estructura básica de HTML5:

<!DOCTYPE html>
<html lang="es">
	<head>
		<meta charset="UTF-8">
		<title>Document</title>
	</head>
	<body>
	</body>
</html>

Elementos de bloque

Los elementos en HTML usualmente son elementos "en bloque" o elementos "en línea". Un elemento en bloque ocupa todo el espacio de su elemento padre (contenedor), creando así un "bloque". Los navegadores suelen mostrar el elemento a nivel de bloque con un salto de línea antes y después del elemento. Los elementos de bloque sólo deben aparecer dentro del elemento <body>.

A continuación vemos algunos ejemplos de etiquetas en bloque.

<article> <aside> <audio> <div> <footer> <h1 a h6> <header> <hr> <li> <main> <nav> <ol> <p> <pre> <section> <table> <ul>

Elementos de línea

Los elementos en HTML usualmente son elementos "en línea" o elementos "en bloque". Un elemento en línea ocupa sólo el espacio delimitado por las etiquetas que definen el elemento en línea. Los elementos de línea sólo deben aparecer dentro del elemento <body>.

A continuación vemos algunos ejemplos de etiquetas en línea.

<a> <abbr> <ancla> <br> <code> <em> <id> <iframe> <img> <ins> <mark> <small> <span> <strong> <sub> <sup> <video>

Formularios

Los formularios HTML son uno de los puntos principales de interacción entre un usuario y un sitio web o aplicación. Ellos permiten a los usuarios enviar información a un sitio web. La mayor parte de las veces se envía información a un servidor web.

Un formulario HTML está hecho de uno o más inputs. Estos inputs puede ser campos de texto (de una linea o multilínea), cajas de selección, botones, checkboxes, o botones de radio. La mayoría del tiempo, estos input están junto a un label que describe su propósito.

Antes de comenzar a escribir código, es siempre mejor dar un paso atrás y tomarse el tiempo de pensar tu formulario. Diseñar un boceto te ayudará a definir el correcto conjunto de información que quieres preguntarle a tu usuario. Desde un punto de vista de experiencia del usuario, es importante recordar que mientras más grande sea tu formulario, mayor será el riesgo de perder usuarios. Mantenlo simple y mantén el foco: pregunta solamente lo que necesitas. Diseñar formularios es un paso importante cuando estés construyendo tu sitio o aplicación.

Todos los formularios HTML comienzan con el elemento <form></form>, este elemento define formalmente un formulario, es un contenedor, puede contener atributos específicos para configurar la forma del formulario. Todos sus atributos son opcionales pero action y method son obligatorios para una buena práctica.

  • El atributo action define la URL desde donde la información recolectada por el formulario debería enviarse.
  • El atributo method define con que método HTTP se enviará la información, puede ser "get" o "post", se recomienda "post".

Los elementos <div> pueden ayudar a estructurar nuestro código de forma conveniente y poder darles estilos de una forma más sencilla. Es obligatorio el uso del atributo for en todos los elementos <label>; es la forma correcta de unir un label a un input en un formulario. Este atributo referencia el id del input correspondiente. Hay algunos beneficios de hacer esto, como permitir al usuario hacer click en el label para activar el input correspondiente.

En el elemento <input>, el atributo mas importante es type. Este atributo es sumamente importante porque define la forma en que el elemento <input> se comporta. Puede cambiar radicalmente el elemento. El valor text, el valor por defecto de este atributo, representa un campo de texto básico de una linea que acepta cualquier tipo de texto sin ningún control o validación.Puede ver la lista de atributos y valores completos aquí.
